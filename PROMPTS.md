**Prompts for the multiplayer mushroom munching game**

# Iteration 1

Code is generated by Claude Sonnet v4.6 high.

Generate a Typescript WebApp multiplayer game (index.html, some ts files and css) which builds with node. The arcade style game is a colorful funny jumping and throw adventure for two competing players. Each player controls a super mario inspired figure (wasd + q and arrow keys + strg on the keyboard to controll the own figure). Game mechanics: Each player tries to hit the other player with mushrooms thrown on q and strg respectively which they trow in a tangent. If hit, there is a small chewing animation and the mushroom is gone. The character grows a little and is easier to hit afterwards. The camera zooms out very slowly such that if players do not hit each other with the mushrooms appear both shrinked relatively to the  view and background while if regularly hit, they (or the one hit) grows in size relatively.
The round ends if no one is hit for 30 seconds (there is a timer) or if one player is hit 30 times over any 30 second intervall (normally because the player got too large and is hit easily therefore). The bigger a player gets, the faster they are throwing. default size can throw a mushroom every 4 seconds. You are a game design expert and UX expert and also a Software Architect expert. Think first, then plan, then implement. Ask open questions in-between. The game is called munchroom, from mushrom and munching. Do not be lazy.
